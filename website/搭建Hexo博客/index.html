配置Hexo

使用docker镜像

根据这篇文章，创建文件docker-composed.yml。

启动docker

docker-compose up -d

进入docker的bash

docker-compose exec hexo bash

配置butterfly主题

  1. 下载主题

  2. 复制配置

  3. 在./_config.butterfly.yml中配置主题

配置github ssh-key

docker使用的ssh为${docker_data}\.ssh，将在github上已配置的ssh-key复制到路径即可。

配置github-page

1. 创建仓库

在github上创建仓库名称为${username}.github.io

2. 配置域名

在hexo目录下，路径source创建CNAME文件，内容为你自己的域名

3. 配置自动发布到github

_config.yml配置

hexo g -d 即可生成静态页面，并上传到github了。

配置Typora

下载数据目录

从github上clone数据目录，并用Typora打开/source/_posts文件夹，进行编辑。

图片路径设置

在偏好设置中，选择图像，复制到指定路径: ../../source/images/${filename}

推荐指定绝对地址${数据目录}/source/images/${filename} ,这样你创建子文件夹进行分类时，也可以正常显示图片。
